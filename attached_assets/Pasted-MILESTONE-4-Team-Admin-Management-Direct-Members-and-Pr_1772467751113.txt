1. TEAM ADMIN MANAGEMENT (Root Admin only)

3. DIRECT MEMBER MANAGEMENT (Team Admin: own team. Root Admin: any team)

   - Creates User (role=MEMBER) + TeamMembership (accessMode=DIRECT, budget, models, periodStart/End)

   - Actions: edit budget/models, suspend, reactivate, remove

4. PROVIDER KEY PROVISIONING — All Three Providers (Section 5.3)

   OpenAI (FULL_AUTO):
   - POST /api/members/[id]/provision
   - Creates project: POST /v1/organization/projects { name: "allotly-{teamSlug}-{memberSlug}" }
   - Creates service account: POST /v1/organization/projects/{projectId}/service_accounts { name: "allotly-managed" }
   - Response includes api_key.value — this is the member's key
   - Save project ID + service account ID + key ID to ProviderMemberLink
   - Show key to member ONCE via KeyRevealCard
   - SetupStatus: PENDING, PROVISIONING, COMPLETE

   Anthropic (SEMI_AUTO):
   - Creates workspace: POST /v1/organizations/workspaces { name: "allotly-{teamSlug}-{memberSlug}" }
   - Save workspace ID to ProviderMemberLink
   - Cannot create key via API — generate setup instructions (markdown) telling member to create key in Anthropic Console
   - SetupStatus: PENDING, PROVISIONING, AWAITING_MEMBER
   - "Mark as Complete" button for when member confirms setup

   Google (GUIDED):
   - No API calls. Generate step-by-step markdown instructions for manual setup.
   - SetupStatus: PENDING, AWAITING_MEMBER
   - "Mark as Complete" button

5. KEY MANAGEMENT
   - POST /api/members/[id]/regenerate-key/[provider] — revoke old key via provider API + create new one
   - POST /api/members/[id]/revoke-key/[provider] — revoke only
   - POST /api/members/[id]/mark-complete/[provider] — mark guided/semi-auto setup as done
   - OpenAI revocation: DELETE /v1/organization/projects/{projectId}/service_accounts/{svcAcctId}
   - Anthropic revocation: List keys GET /v1/organizations/api_keys?workspace_id={id}, then POST each to inactive

6. PERMISSION ENFORCEMENT (Section 7.2)
   - Every API route checks orgRole
   - Team Admins can only access their own team's members
   - Root Admin can access any team
   - Members cannot access these routes at all
