We shipped Allotly MCP V1 (commit 29ec1ab). It's mounted at /mcp,
13 tools, all tests green. Read server/lib/mcp/README.md for full
context.

V1 onboarding doesn't work for Claude Desktop or claude.ai users
because those hosts only support stdio transport. This milestone
ships a thin npm package that bridges stdio JSON-RPC to our hosted
HTTP /mcp endpoint, so a one-line config in Claude Desktop or
Cursor's stdio mode "just works."

    Standalone, ESM, target ES2022, outDir dist, strict on.

  packages/allotly-mcp/src/index.ts
    Hashbang #!/usr/bin/env node at line 1.
    - Read ALLOTLY_KEY from process.env. If missing or empty, write a
      friendly multi-line error to stderr (explaining where to set it,
      with examples for Claude Desktop and Cursor) and exit(1).
    - Read ALLOTLY_MCP_URL from env, default https://allotly.ai/mcp.
    - Spawn `mcp-remote` (programmatic API preferred; fall back to
      child_process spawn if mcp-remote only ships a CLI). Pass:
        url:    ALLOTLY_MCP_URL
        header: `Authorization: Bearer ${ALLOTLY_KEY}`
    - Pipe stdin/stdout through. stderr forwards but with an
      "[allotly-mcp]" prefix so users can tell our log lines from
      mcp-remote's.
    - Forward SIGINT/SIGTERM to the child.
    - On child exit, exit with the child's code.

  packages/allotly-mcp/README.md
    Standalone install instructions for Claude Desktop, Cursor,
    Claude Code, and VS Code. This file is what shows on npmjs.com,
    so make it self-explanatory — don't assume the reader knows what
    Allotly is. Pull tone from server/lib/mcp/README.md but rewrite
    for an external-package audience.

ACCEPTANCE:

  - `cd packages/allotly-mcp && npm install && npm run build`
    produces dist/index.js with executable bit set, zero TypeScript
    errors under strict mode.
  - `ALLOTLY_KEY= node dist/index.js` (empty key) prints the friendly
    error and exits 1.
  - `ALLOTLY_KEY=allotly_sk_test node dist/index.js < /dev/null`
    starts, attempts to connect, exits when stdin closes.
  - `npx vitest run` inside packages/allotly-mcp passes.
  - Linked install works: from packages/allotly-mcp run `npm link`,
    then from a temp directory `npm link allotly-mcp` (or the scoped
    name) and execute `allotly-mcp`.
