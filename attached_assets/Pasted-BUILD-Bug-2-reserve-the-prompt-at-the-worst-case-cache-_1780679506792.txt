BUILD — Bug 2: reserve the prompt at the worst-case cache-write rate (Option A). Green light to implement.

What & why. A member can overshoot their budget cap on cached prompts. The pre-flight reservation prices the prompt at the base input rate (1×), but settlement prices Anthropic cache_creation_input_tokens at 1.25× (CACHE_WRITE_MULTIPLIER). When the prompt is large and max_tokens is small (little output slack), settled prompt cost exceeds what was reserved, and the post-call adjustment debits the shortfall after the request already ran — so the member exceeds their cap. Fix: hold the prompt at 1.25× in the reservation. Since 1.25× ≥ 1× ≥ 0.1×, this guarantees reserved ≥ settled for any cache mix. Settlement is untouched, so the actual billed/displayed amount never changes — the extra hold is refunded the instant real usage settles.
