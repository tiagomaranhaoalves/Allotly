BUG #1 — gpt-5 family is sending max_tokens instead of max_completion_tokens
================================================================

File: server/lib/proxy/translate.ts:158
Current code:
    const isReasoningModel = /^(o1|o3|o4)/.test(azureContext.modelId);

Problem: This regex catches o1/o3/o4 but NOT gpt-5*. So gpt-5, gpt-5-mini, gpt-5.1, gpt-5.2, gpt-5.2-chat fall into the else branch on line 165 which sets body.max_tokens. Azure rejects with: "Unsupported parameter: 'max_tokens' is not supported with this model. Use 'max_completion_tokens' instead."

Direct proof: I called the user's APIM directly at /openai/deployments/gpt-5/chat/completions?api-version=2024-10-21 with ONLY max_completion_tokens in the body — Azure returned HTTP 200 with a real completion. So gpt-5 family on the user's APIM works fine when given max_completion_tokens.

Fix: Update the regex on translate.ts:158 to also match gpt-5:
    const isReasoningModel = /^(o1|o3|o4|gpt-5)/.test(azureContext.modelId);

ALSO check translate.ts:139 (the same regex pattern in the OPENAI branch). Update both for consistency, since OpenAI's gpt-5 family will eventually have the same constraint.
