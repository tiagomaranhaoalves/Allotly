BUG 1 — Sub-cent settlement leak (compounding, unbounded).
Settlement computes the real cost then rounds to whole cents (Math.round(trueCost)). BUG 2 — Cache-write under-reservation (bounded, self-correcting).
Reservation prices the prompt at the base input rate (1×), but settlement prices Anthropic cache_creation_input_tokens at 1.25×. When the prompt is large and max_tokens is small (little output slack), settlement exceeds the reservation, and the post-call adjustment debits the shortfall after the request already succeeded — so a member can overshoot their cap by that gap.

Worked example: Claude at $3/M input, $15/M output, 200k-token prompt, max_tokens=100. Reserved = ceil(200000×300/1e6)=60¢ + ceil(100×1500/1e6)=1¢ = 61¢. Settled (first call writes whole prompt to cache) = round(200000×300×1.25/1e6) = 75¢. Under-reserved by 14¢ → member overshoots by 14¢. (Cache reads at 0.1× only make settlement cheaper, so they're already safe — this is purely cache-write.)
