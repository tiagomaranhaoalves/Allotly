DECISIONS — confirm before coding:

  D1. JWT signing. Use HMAC-SHA256 with a 256-bit secret read from
      env OAUTH_JWT_SECRET. Generate one fresh secret if missing
      (log a warning, don't crash). Access tokens carry
      { iss: "https://allotly.ai", sub: <userId>, aud: <resource>,
        membership_id, client_id, scope, jti, iat, exp } and expire
      in 15 minutes.

  D2. Refresh tokens. Opaque 256-bit random strings, stored as
      sha256 hashes in oauth_tokens.refresh_token_hash. Rotated on
      every use (RFC 6749 §10.4 — issue a new refresh token,
      revoke the old one in the same transaction). Refresh expires
      in 30 days.

  D3. Audit log identity. Switch the existing principalHash
      computation in mcp/auth.ts to:
        bearerKind === "key"     -> "key:" + apiKeyId
        bearerKind === "voucher" -> "voucher:" + sha256(voucherCode)
        bearerKind === "oauth"   -> "oauth:" + clientId + ":" + userId
      This stays stable across token rotations. Add client_id and
      audience columns to mcp_audit_log so per-app traces are
      possible. The old principalHash=sha256(bearer) goes away.

CRYPTO REQUIREMENTS:

  - Authorization codes: 32 random bytes (base64url), stored hashed
    (sha256), expires_at 60 seconds, consumed_at single-use atomic.
  - PKCE: S256 only, reject any code_challenge_method !== "S256".
    Verifier comparison via crypto.timingSafeEqual.
  - JWT: HS256 only, OAUTH_JWT_SECRET ≥ 32 bytes, validated with
    explicit alg whitelist (defend against "alg: none").
  - Refresh tokens: 32 random bytes, stored hashed (sha256), rotated
    on every use, old hash revoked in the same transaction.
  - Client secrets (for confidential clients only — most MCP clients
    are public so client_secret will be null): bcrypt at cost 12.

DCR DETAILS (RFC 7591):

  POST /oauth/register
    body: { client_name, redirect_uris[], grant_types?,
            response_types?, scope?, token_endpoint_auth_method? }
    response: { client_id, client_secret?, client_id_issued_at,
                client_secret_expires_at: 0, registration_access_token,
                registration_client_uri, ...the request echoed back }
  No pre-approval. Rate-limit: 10 registrations per IP per hour.
  redirect_uris validated against a regex (no localhost in prod
  except for development build).

CONSENT SCREEN (server-rendered HTML, NO client JS):

  Inputs from the validated /oauth/authorize: client_name, scopes,
  current user (from session). Renders:
    "claude.ai is requesting access to your Allotly account
     (tiagomaranhaoalves14nov@gmail.com). It will be able to:
       ✓ send chat messages on your behalf and consume your budget
       ✓ view your remaining budget, status, and recent usage
     [Approve]  [Deny]"
  POST /oauth/consent with the form's hidden fields (auth request
  ID, csrf token from the session) issues the code or returns a
  denied error to the redirect_uri. CSP: default-src 'self'; no
  scripts; no images outside our domain.
