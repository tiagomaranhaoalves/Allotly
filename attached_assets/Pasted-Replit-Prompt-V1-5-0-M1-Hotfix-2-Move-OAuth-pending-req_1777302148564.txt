Replit Prompt — V1.5.0 M1 Hotfix #2: Move OAuth pending-request store to Redis
You are fixing a production bug encountered live on prod just now: a real OAuth flow from Claude.ai successfully reached the Allotly consent screen, the user clicked "Authorize", and the server returned auth_request expired or unknown. The cause is server/lib/oauth/authorize.ts:27 — the pending Map is in-memory, so when the consent POST lands on a different instance (or the same instance after a cold restart), the entry is missing and consent fails.

Locked decisions — implement, do not ask. One end-of-task report.

Locked decisions
Storage: Redis. Reuse the existing redisSet / redisGet / redisDel helpers in server/lib/redis.ts. Do NOT introduce a new client.
Key namespace: allotly:oauth:pending:<nonce>. Match the existing pattern (allotly:oauth:revoked:<jti> etc.).
TTL: 300 seconds (matches PENDING_TTL_MS = 5 * 60 * 1000). Redis TTL replaces the in-process pruneExpired() call, which is no longer needed.
Value: JSON-serialized PendingAuthRequest (the existing interface — clientId, redirectUri, scope, resource, state, codeChallenge, expiresAt, userId). Just JSON.stringify on write, JSON.parse on read.
Single-use: consent must atomically read-and-delete the entry to prevent double-submission. Use Redis GETDEL if available in our Redis client; otherwise GET then DEL. Add a comment noting the race window if non-atomic.
Failure mode: if Redis is down, redisSet will throw — let it. Do NOT silently fall back to in-memory. The user will see a 5xx instead of a confusing "expired" error.
No DB changes. No db:push.
Do NOT touch: anything outside server/lib/oauth/authorize.ts and the new tests.

const PENDING_KEY = (nonce: string) => `allotly:oauth:pending:${nonce}`;
async function setPending(nonce: string, p: PendingAuthRequest): Promise<void> {
  await redisSet(PENDING_KEY(nonce), JSON.stringify(p), 300);
}
async function takePending(nonce: string): Promise<PendingAuthRequest | null> {
  // GETDEL = atomic read-and-delete to prevent consent double-submission
  const raw = await redisGetDel(PENDING_KEY(nonce));
  return raw ? (JSON.parse(raw) as PendingAuthRequest) : null;
}

Redis helper
server/lib/redis.ts — add redisGetDel(key) if missing. Implementation:

export async function redisGetDel(key: string): Promise<string | null> {
  // Atomic read-and-delete. Falls back to GET+DEL on older Redis (<6.2) but that
  // race window is acceptable here — consent double-submission is rare and the
  // worst case is "auth_request expired or unknown" on the second click.
  const client = await getClient();
  if (typeof (client as any).getdel === "function") {
    return (await (client as any).getdel(key)) ?? null;
  }
  const v = await client.get(key);
  if (v !== null && v !== undefined) await client.del(key);
  return v ?? null;
}
