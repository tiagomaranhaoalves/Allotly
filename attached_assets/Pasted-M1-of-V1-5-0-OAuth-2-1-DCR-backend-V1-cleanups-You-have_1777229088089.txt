ENDPOINTS (server/lib/oauth/)

  GET  /.well-known/oauth-authorization-server  — RFC 8414 metadata.
       issuer = https://allotly.ai. Lists authorize/token/register/
       revoke endpoints, scopes_supported = ["mcp", "mcp:read"],
       grant_types = ["authorization_code","refresh_token"],
       response_types = ["code"], code_challenge_methods = ["S256"]
       only, token_endpoint_auth_methods = ["none","client_secret_basic"].

  POST /oauth/register  — RFC 7591 DCR. Body { client_name,
       redirect_uris[], grant_types?, response_types?, scope?,
       token_endpoint_auth_method? }. No pre-approval. Rate limit
       10/IP/hour. Validate redirect_uris regex (block localhost
       except in dev). Returns { client_id, client_secret? (null
       for public clients), client_id_issued_at,
       client_secret_expires_at: 0, registration_access_token,
       registration_client_uri, ...echoed metadata }. Stores in
       oauth_clients with bcrypt(cost=12) on client_secret if set.

  GET  /oauth/authorize  — Validates client_id, redirect_uri (must
       match a registered URI), state, code_challenge,
       code_challenge_method = "S256" (reject "plain" outright),
       response_type = "code", scope subset of registered scopes,
       resource (RFC 8707) MUST equal "https://allotly.ai/mcp" or
       reject. If session unauthenticated, redirect to
       /login?next=<this-url>. If users.isVoucherUser = true, redirect
       to /oauth/claim-account?next=<this-url>. Otherwise render the
       consent template.

  POST /oauth/consent  — Receives form { auth_request_id, csrf, decision }.
       Validates csrf against session. On approve: insert into
       oauth_authorization_codes (code = 32 random bytes base64url,
       stored as sha256, expires 60s, consumed_at NULL,
       code_challenge, redirect_uri, scope, resource, membership_id),
       redirect to redirect_uri with ?code=<plaintext>&state=<echoed>.
       On deny: redirect with ?error=access_denied&state=<echoed>.

  POST /oauth/token  — Two grant types:
         (a) authorization_code: validate code via atomic
             UPDATE oauth_authorization_codes
               SET consumed_at = NOW()
               WHERE code_hash = sha256(:code)
                 AND consumed_at IS NULL
                 AND expires_at > NOW()
               RETURNING …
             Compare PKCE verifier with crypto.timingSafeEqual against
             stored code_challenge (S256). On match: issue access
             (15min HS256 JWT { iss, sub: userId, aud: resource,
             membership_id, client_id, scope, jti, iat, exp }) and
             refresh (32 random bytes, sha256 stored). Insert into
             oauth_tokens in same transaction.
         (b) refresh_token: lookup by sha256, atomic mark-revoked,
             issue new access + new refresh in same transaction.
             Reject if old refresh was already revoked (replay
             defence).
       Reject any unknown grant_type with 400 unsupported_grant_type.

  POST /oauth/revoke  — RFC 7009. Body { token, token_type_hint }.
       Set revoked_at on the matching row. 200 always (don't leak
       token existence). Add jti to Redis revoked set
       (allotly:oauth:revoked:{jti}, TTL = remaining token lifetime)
       so /mcp checks reject immediately without DB roundtrip.

CRYPTO REQUIREMENTS

  - Authorization codes: crypto.randomBytes(32).toString("base64url"),
    stored as sha256 hash, 60s expiry, atomic single-use.
  - PKCE: S256 only. Verifier comparison via crypto.timingSafeEqual.
    Reject code_challenge_method !== "S256" at /oauth/authorize.
  - JWT: HS256 with explicit alg whitelist (defend against alg=none
    attack). Validate iss, aud, exp, jti-not-revoked.
  - Refresh tokens: 32 random bytes, stored sha256, rotated atomically
    on every use.
  - Client secrets: bcrypt cost 12. Public clients (no secret) are
    the norm for MCP — secret is null.
