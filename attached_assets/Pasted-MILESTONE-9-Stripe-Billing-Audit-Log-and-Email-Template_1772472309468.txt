1. STRIPE SUBSCRIPTION BILLING (Section 15)
   - Product 1: "Allotly Team — Team Admin Seat" at $20/month, quantity-based
   - POST /api/billing/checkout — create Stripe Checkout Session
     - For new subscribers: full checkout
     - For existing: update subscription quantity (+1 seat per new Team Admin)
   - POST /api/billing/portal — create Stripe Customer Portal session
   - GET /api/billing/subscription — current subscription details
   - POST /api/billing/webhooks — Stripe webhook handler (Section 15.3):
     - checkout.session.completed (subscription): upgrade to TEAM, set maxTeamAdmins
     - checkout.session.completed (bundle): create VoucherBundle, init Redis counters (already built in M7, wire it up here)
     - customer.subscription.updated: sync seat count, suspend excess if decreased
     - customer.subscription.deleted: downgrade to FREE with 30-day grace
     - invoice.payment_failed: email Root Admin, 7-day grace
   - Settings page (/dashboard/settings): plan details, "Upgrade to Team" button, "Manage Billing" link to Stripe Portal
   - Plan enforcement on every mutation: check member limits, voucher limits, provider limits per plan
- Bundle purchases are NOT gated by plan. Free plan users can purchase External Access Bundles. When creating vouchers from a bundle, the bundle limits apply regardless of plan. Only plan-included voucher allowances use the Free tier limits.
