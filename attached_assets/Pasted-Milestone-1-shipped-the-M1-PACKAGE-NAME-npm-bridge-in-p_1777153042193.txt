LAYOUT (top to bottom, single column on mobile):

  Header:
    Title: connect.title  ("Connect Allotly to your AI tool")
    Subtitle: connect.subtitle  ("Pick your tool, copy the config,
              paste it in. Tested in under a minute.")

  Key selector:
    Dropdown of the user's active keys, labelled by their keyPrefix.
    Default selection: most recently used key (or the one specified
    in ?key= query param).

  Mask toggle:
    An eye icon toggle in the top-right of the connector grid. When
    masked, every snippet shows allotly_sk_•••••<last4>. Default OFF
    on this page (returning users — they should reveal explicitly).

  Connector grid (2x2 on desktop, stacked on mobile). Each card:
    - Tool logo (use lucide icons or simple inline SVGs).
    - Heading + 1-line "Best for: ..." subtitle.
    - Code block with the snippet, showing the user's selected key.
    - Copy button (use navigator.clipboard.writeText, with a 2-second
      "Copied!" confirmation state).
    - "Open in <tool>" link if a documented deep-link scheme exists
      (cursor:// for Cursor, vscode:// for VS Code; verify, omit if
      not documented).

  Snippets per card (substitute <KEY> with the selected key):

    Cursor (~/.cursor/mcp.json):
      { "mcpServers": { "allotly": {
          "url": "https://allotly.ai/mcp",
          "headers": { "Authorization": "Bearer <KEY>" } } } }

    VS Code (.vscode/mcp.json):
      { "servers": { "allotly": {
          "url": "https://allotly.ai/mcp",
          "type": "http",
          "headers": { "Authorization": "Bearer <KEY>" } } } }

    Claude Code (terminal):
      claude mcp add --transport http allotly https://allotly.ai/mcp \
        --header "Authorization: Bearer <KEY>"

    Claude Desktop (claude_desktop_config.json):
      { "mcpServers": { "allotly": {
          "command": "npx",
          "args": ["-y", "<M1 PACKAGE NAME>@latest"],
          "env": { "ALLOTLY_KEY": "<KEY>" } } } }
      Caption under this card: "Uses the <M1 PACKAGE NAME> bridge —
      auto-installs on first run, no extra setup."

  Test connection panel:
    Button labelled "Test connection." On click:
      POST https://allotly.ai/mcp
      headers: { Authorization: "Bearer <selected-key>",
                 "Content-Type": "application/json" }
      body: { jsonrpc: "2.0", id: 1, method: "tools/list" }
    Render one of three states:
      - Green: "✓ Connected — N tools available" (use the actual
        count from the response).
      - Red: "Key invalid or revoked." (on JSON-RPC error code
        -32001 or HTTP 401)
      - Yellow: "Couldn't reach allotly.ai/mcp" (on network failure).
    Show a "Show response" expandable with the raw JSON for debugging.

  "What can I do once connected?" expandable:
    Three example prompts users can paste into their tool:
      - "Use the allotly MCP to list my available models."
      - "Compare gpt-4o-mini and claude-haiku-4-5: what's the
         difference between TCP and UDP?"
      - "How much budget do I have left?"
